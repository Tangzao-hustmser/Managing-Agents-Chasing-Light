# 命题 D 头脑风暴与优化路径（2026-03-31）

## 1. 当前优势（可用于答辩强调）

- 已形成完整业务主链路：申请 -> 审批 -> 扣减库存 -> 归还/异常处理。
- 智能体支持“确认后执行”，不是纯问答机器人。
- 具备调度推荐、增强分析、公平性与异常评分等治理能力。
- 支持证据流（Qiniu + 盘点入口），具备可追溯性基础。

## 2. 关键不足（按影响排序）

1. 异常处理没有完整“任务闭环”操作入口。
2. 智能体对闭环任务缺少可执行动作，治理链路停留在建议层。
3. 前端未突出“异常治理结果”，演示时难体现管理闭环价值。
4. 文档虽丰富，但“竞赛评分点 -> 系统能力映射”不够集中。

## 3. 可选优化方向（发散）

- A. 异常闭环任务中心（任务状态、负责人、处理记录、超期跟踪）。
- B. 智能体闭环工具化（查询待办、确认执行完成/处理中）。
- C. 资源预约公平策略（高频占用用户限流、黄金时段配额）。
- D. 盘点证据智能化（OCR/视觉模型识别、自动差异工单）。
- E. 通知协同（飞书/邮件提醒超时归还和待办任务）。

## 4. 最优路径（本次采纳）

优先落地 A + B + C（轻量前端呈现），原因：

- 直接补齐“治理闭环”短板，贴合命题 D 的核心痛点（浪费、丢失、占用不均）。
- 复用现有数据模型 `FollowUpTask`，改造成本低、可快速验收。
- 对评审最直观：可展示从异常发现到任务完成的全流程。

## 5. 本次已实施项

- 新增闭环任务接口：
  - `GET /follow-up-tasks`
  - `PATCH /follow-up-tasks/{task_id}`
- 智能体新增闭环任务能力：
  - 查询闭环任务摘要
  - 通过确认链路执行“任务处理中/任务完成”
- 前端新增“异常闭环任务”卡片，支持状态更新。
- 新增自动化测试 `tests/test_follow_up_tasks.py` 覆盖权限与 Agent 执行链路。

## 6. 下一步建议（可继续加分）

1. 加入“任务超期升级”规则（超期自动提升告警级别）。
2. 引入消息推送（飞书/邮件）形成主动提醒。
3. 为 `FollowUpTask` 增加 `updated_at/closed_at/result` 字段，增强审计粒度。
4. 增加“资源公平调度策略”配置化开关（竞赛创新点更突出）。
