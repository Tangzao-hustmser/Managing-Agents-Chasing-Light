## 🔐 安全性说明

**认证版的安全特性**：

1. **令牌存储**
   ```javascript
   // 浏览器 localStorage 中存储 token
   localStorage.setItem('token', data.token);
   localStorage.setItem('user', JSON.stringify(data.user));
   ```

2. **权限检查**
   - 每个 API 请求都验证 `Authorization` header
   - 后端检查用户角色和权限
   - 无权限的数据不返回

3. **会话管理**
   - 登录后保存 token
   - 关闭浏览器或 24 小时后自动过期（演示模式）
   - 可点"退出登录"立即清除

## 📝 技术细节

### 登录流程

```javascript
1. 用户输入账号密码
   ↓
2. POST /auth/login
   ↓
3. 后端验证，返回 token + 用户信息
   ↓
4. 前端保存到 localStorage
   ↓
5. 跳转到 /dashboard-main
   ↓
6. 所有 API 请求自动带 Authorization header
   ↓
7. 后端验证 token，返回权限范围内的数据
```

### 权限控制

```python
# 后端示例（routers/transactions.py）

def get_transactions(db, current_user):
    if current_user.role == 'admin':
        # 管理员看全部
        return db.query(Transaction).all()
    else:
        # 学生/教师只看自己的
        return db.query(Transaction).filter(
            Transaction.user_id == current_user.id
        ).all()
```

## 💡 常见问题

**Q: 两个版本可以同时用吗？**
A: 完全可以。演示版 `/dashboard` 用于快速展示，认证版 `/login` 用于完整功能演示。

**Q: 演示版的数据和认证版一样吗？**
A: 是的，都连同一个数据库，数据实时同步。

**Q: 如何修改演示账户？**
A: 编辑 `app/seed.py`，重新运行 `python -m app.seed` 即可。

**Q: 登录失败怎么办？**
A: 
1. 检查账户名和密码是否正确
2. 确保服务启动：终端显示 "Application startup complete"
3. 尝试清除浏览器缓存

**Q: 如何退出登录？**
A: 点击右上角"退出登录"按钮，返回登录页。
